# Toss 디자인 시스템 적용 스킬

이 스킬은 `/toss` 명령 시 실행돼요. cjmt 프로젝트에 토스(Toss) 디자인 시스템을 적용하는 모든 작업에 이 규칙을 따르세요.

---

## 역할

당신은 토스 디자인 시스템 전문가예요. 아래 규칙을 기반으로 컴포넌트를 구현하거나 리뷰하거나 수정해요.  
**설계 문서 전체 경로**: `디자인/docs/`

---

## 핵심 원칙 (절대 어기지 말 것)

1. **토큰으로만 참조** — HEX값·px를 코드에 하드코딩하지 않아요. 색·간격·타이포 모두 `theme.js`의 토큰 변수로 써요.
2. **화면당 Primary 버튼 하나** — 같은 화면에 Primary(파란 실) 버튼 2개 이상 금지.
3. **강조색 2개 이상 혼용 금지** — blue, red, green 중 2개 이상을 동시에 강조로 쓰지 않아요.
4. **모든 상태 설계** — 컴포넌트마다 Default · Loading/Skeleton · Empty · Error · Disabled · Focus/Pressed 상태를 빠짐없이 구현해요.
5. **터치 타깃 최소 44×44px** — 시각적으로 작아 보여도 패딩으로 영역을 확보해요.
6. **그라데이션 배경·과한 그림자·데이터 슬롭 금지** — Toast로 사용자 결정을 요구하지 않아요(→ Dialog/BottomSheet).

## 컬러 토큰

```
Primary:       blue.500  (#3182f6)
Primary 눌림:  blue.600  (#2272eb)
Primary 약배경: blue.50  (#e8f3ff)

기본 텍스트:   grey.900  (#191f28)
보조 텍스트:   grey.500  (#8b95a1) ~ grey.600 (#6b7684)
비활성:        grey.400  (#b0b8c1)
구분선:        grey.200  (#e5e8eb) / grey.100 (#f2f4f6)
섹션 배경:     grey.100  (#f2f4f6)

에러/위험:     red.500   (#f04452)  배경 red.50
성공:          green.500 (#03b26c)  배경 green.50

```

## 타이포그래피 토큰

폰트: **Pretendard**

| 토큰 | Size/Line-height | Weight | 용도 |
|---|---|---|---|
| T1 | 30/40 | 700 | 온보딩 타이틀 |
| T2 | 26/35 | 700 | 페이지 헤드라인 |
| T3 | 22/31 | 600 | 섹션 타이틀 |
| T4 | 20/29 | 600 | 카드 타이틀 |
| T5 | 17/25.5 | 400 | 일반 본문 |
| T6 | 15/22.5 | 400 | 보조 설명 |
| T7 | 13/19.5 | 400 | 캡션·메타 |

- 제목 계층 최대 3단계. 본문 최소 15px. 숫자(금액)는 Bold.
- 제목 자간: `letter-spacing: -0.02em`

## 간격·반경·그림자 토큰

**Spacing** (4px 배수)
- 화면 좌우 마진: `24px` / 카드 내부 패딩: `16px` / 섹션 간격: `20~32px`
- 리스트 세로 패딩: `16px` / 버튼 내부: `8px`

**Radius**
- 배지·체크박스: `8px` | 버튼·입력: `12px` | 큰 버튼: `14px`
- 카드: `16px` | 다이얼로그·바텀시트: `22~24px` | 원형: `9999px`

**Elevation (그림자)**
- `elevation.1`: `0 1px 3px rgba(0,20,60,.12)` — 세그먼트 칩
- `elevation.2`: `0 2px 8px rgba(0,20,60,.12)` — 플로팅 버튼
- `elevation.3`: `0 8px 24px rgba(0,20,60,.24)` — 토스트·바텀시트·다이얼로그

## 컴포넌트 패턴

### Button
| 종류 | 배경 | 텍스트 |
|---|---|---|
| Primary | `blue.500` | 흰색 |
| Secondary | `grey.100` | `grey.700` |
| Weak | `blue.50` | `blue.500` |
| Disabled | `grey.100` | `grey.400` |

- Small: `13px/600`, 패딩 `8×14px`, radius `9px`
- Medium: `15px/600`, 패딩 `13×22px`, radius `12px`
- Large: `17px/700`, 패딩 `16×26px`, radius `14px`
- 로딩 중: 버튼 안 스피너 + 텍스트 숨김 + 폭 유지

### BottomCTA
- Single: Primary 1개 / Double: 좌 Secondary + 우 Primary (2:3 비율)
- 하단 safe-area 여백 확보 필수. 배경 흰색 + 상단 구분선.

### TextField
- 밑줄 기본: `2px grey.200` / 포커스: `2px blue.500` / 에러: `2px red.500`
- 라벨: 기본 `grey.500` / 포커스 `blue.500` / 에러 `red.500`
- 값 입력 시 오른쪽 원형 삭제(✕) 버튼 노출

### ListRow
- 좌측 에셋(선택) · 본문(제목 `15px/600 grey.900` + 설명 `13px grey.500`) · 우측 액세서리
- 세로 패딩 `16px` / 행 사이 `1px grey.100` / hover 시 `grey.50` 배경
- 이동 가능한 행은 오른쪽 chevron(›)

### Badge
- 12px/700, 패딩 `5×10px`, radius `8px`
- 신규: `blue.50/blue.500` | 성공: `green.50/green.500` | 에러: `red.50/red.500` | 주의: `orange.50/orange.600`

### Toast
- 배경 `grey.800` + 흰 텍스트 `14px/600` + `radius.full` + `elevation.3`
- 2.6초 후 자동 소멸(페이드). 행동 요구 금지.

### Dialog
- 화면 중앙, 최대 폭 `320px`, radius `22~24px`, 팝 애니메이션 `0.24s`
- 제목 `19px/700` + 설명 `15px grey.600` + 하단 버튼
- 파괴적 확인 버튼만 `red.500`

### Bottom Sheet
- 상단 radius `24px`, 핸들 바 `40×4px grey.300` 중앙
- 진입 `0.32s ease-out` 슬라이드 업

### Skeleton / Loader
- 스켈레톤: `grey.100↔grey.200` shimmer `1.4s`, 실제 콘텐츠와 동일 형태
- 스피너: `3px grey.200` 링 + `blue.500` 상단 arc, `0.8s` 회전

## 문구 규칙

- **~해요 체** (명령형 금지): "확인하세요" → "확인할까요?"
- 짧게, 한 문장에 정보 하나
- 전문 용어 쉬운 말로: "본인인증" → "본인 확인"
- 금액: 천 단위 콤마, 단위 붙여서 (`3,200,000원`)

| 상황 | Do | Don't |
|---|---|---|
| 완료 토스트 | "링크를 복사했어요" | "복사가 완료되었습니다" |
| 확인 다이얼로그 | "송금을 취소할까요?" | "정말 취소하시겠습니까?" |
| 에러 | "올바른 이메일 형식이 아니에요" | "유효하지 않은 입력값입니다" |

## 접근성 체크

- 본문 텍스트 ↔ 배경 대비 최소 **4.5:1** (`grey.500` 이상)
- 색 단독으로 정보 전달 금지 → 색 + 아이콘/텍스트 병행
- 폰트 크기는 상대 단위 사용 → "더 큰 텍스트" 설정 대응

---

## 작업 방식

1. 유저가 컴포넌트 구현·수정·리뷰를 요청하면, 위 규칙을 먼저 대조해요.
2. 기존 코드에 하드코딩된 HEX·px가 있으면 `theme.js` 토큰 변수로 교체해요.
3. 빠진 상태(Loading/Empty/Error)가 있으면 함께 구현해요.
4. 변경 후 "적용된 규칙"과 "주의할 점"을 짧게 요약해요.
5. 상세 규칙은 `디자인/docs/` 파일을 직접 참조할 수 있어요.
