# Feedback — 피드백 계열

상태·진행·결과를 알려주는 컴포넌트들.

---

## Toast

화면 하단에 잠깐 떠서 결과를 알리는 알약형 알림.
- 배경 `grey.800`, 흰 텍스트 14px/600, `radius.full`, `elevation.3`.
- 성공은 앞에 초록 체크 아이콘. 약 2.6초 후 자동으로 사라져요(페이드).
- **행동을 요구하지 않는** 가벼운 피드백에만 사용. 선택이 필요하면 Dialog/BottomSheet를 써요.

## Progress Bar

작업 진행률을 가로 막대로. 트랙 `grey.200`(높이 8px, `radius.full`), 채움 `blue.500`. 불확정 상태는 좌우로 흐르는 애니메이션.

## Progress Stepper

여러 단계 중 현재 위치. 완료 단계는 `blue.500` + 체크, 현재는 `blue.500` + 숫자, 이후는 `grey.200` + `grey.400` 숫자. 단계 사이 연결선도 진행 색으로.

## Loader

로딩 인디케이터.
- **스피너**: `3px grey.200` 링 + `blue.500` 상단 호(arc), 0.8s 회전.
- **닷(dot)**: 점 3개가 순차로 밝아지는 형태. 인라인 소형 로딩에.

## Skeleton

콘텐츠가 로드되기 전 자리를 잡아주는 회색 블록. `grey.100`↔`grey.200` shimmer 애니메이션(1.4s). 실제 콘텐츠와 **같은 모양/크기**로 만들어 레이아웃이 튀지 않게 해요.

## Result

작업 완료/실패 전체 화면. 중앙 일러스트(또는 아이콘) + 제목 + 설명 + 하단 CTA.
- 성공: `green.500` 계열, 실패: `red.500` 계열 아이콘.

## Tooltip

요소에 붙는 짧은 설명. 배경 `grey.900` + 흰 텍스트 13px, 꼬리로 대상을 가리켜요. 마우스 오버(웹) 또는 최초 진입 시 1회 노출.

## Bottom Info

화면 하단의 보조 안내 문구. 12~13px `grey.500`, 약관·유의사항·환율 기준 같은 부가 정보를 담아요.
