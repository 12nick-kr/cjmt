# Overlay — 오버레이 & 내비게이션 계열

기존 화면 위에 떠서 상호작용을 요구하거나, 화면을 전환하는 컴포넌트들.
모든 오버레이는 뒤에 딤(dim) `greyOpacity` 46% 배경을 깔고, 딤을 탭하거나 아래로 스와이프하면 닫혀요.

---

## Bottom Sheet

화면 하단에서 올라오는 시트. 선택지가 여럿이거나, 맥락을 유지한 채 추가 입력을 받을 때 써요.
- 상단 모서리 `radius.2xl`(24px), 중앙에 40×4px 핸들 바.
- 진입: 0.32s ease-out 슬라이드 업. 닫힘: 아래로 슬라이드.
- 콘텐츠 하단에 BottomCTA(Single/Double)를 붙일 수 있어요.
- **useBottomSheet** 훅으로 열고 결과를 `await`로 받는 패턴 권장.

## Modal

전체를 덮는 큰 오버레이. 독립적인 하위 플로우(예: 상세 설정)를 담아요. 상단에 닫기(✕) 또는 뒤로 가기. 모바일에선 전체 화면에 가깝게.

## Dialog

집중이 필요한 짧은 확인창. 화면 중앙에 뜨고(팝 애니메이션 0.24s), 폭 최대 320px, `radius.2xl`.

| 종류 | 버튼 | 용도 |
|---|---|---|
| **AlertDialog** | 확인 1개 | 단순 알림 (되돌릴 수 없음) |
| **ConfirmDialog** | 취소 + 확인 2개 | 사용자 결정이 필요할 때 |

- 제목 19px/700 + 설명 15px `grey.600` + 하단 버튼.
- 파괴적 행동(삭제 등)의 확인 버튼만 `red.500`로.
- **useDialog** 훅으로 결과(확인/취소)를 `await`.

## Menu

특정 요소에서 펼쳐지는 액션 목록(더보기 ⋯). 각 항목은 44px 높이, 파괴적 항목은 `red.500` 텍스트. `floatedBackground` + `elevation.3`.

---

## Tab

같은 층위 화면 사이를 전환. 선택 탭은 텍스트 `grey.900`/700 + 하단 2px 인디케이터, 비선택은 `grey.400`/500.
- 필터 전환이 아니라 **콘텐츠 전환**일 때 사용(필터는 Segmented Control).

## Top (App Bar)

화면 최상단 바. 좌측 뒤로가기/타이틀, 우측 액션(Icon Button). 스크롤 시 타이틀이 축소되거나 구분선이 나타나는 패턴을 쓸 수 있어요. 높이 56px 기준.
