# Action — 버튼 계열

사용자의 행동을 유도하는 컴포넌트들. 한 화면에서 **가장 중요한 행동 하나만 Primary**로 강조해요.

---

## Button

### 위계 (Hierarchy)
| 종류 | 배경 | 텍스트 | 용도 |
|---|---|---|---|
| **Primary** | `blue.500` | 흰색 | 화면의 핵심 행동 (다음, 확인, 송금) |
| **Secondary** | `grey.100` | `grey.700` | 보조 행동 (취소, 뒤로) |
| **Weak** | `blue.50` | `blue.500` | 가벼운 강조 (더 알아보기) |
| **Disabled** | `grey.100` | `grey.400` | 비활성 (조건 미충족) |

- Hover/Pressed: 배경을 한 단계 진하게 (`blue.500` → `blue.600`).

### 크기 (Size)
| 크기 | 폰트 | 패딩 | 반경 |
|---|---|---|---|
| Small | 13px / 600 | 8×14px | 9px |
| Medium | 15px / 600 | 13×22px | 12px |
| Large | 17px / 700 | 16×26px | 14px |

### 로딩 상태
비동기 처리 중에는 버튼 안에 스피너를 넣고 텍스트를 숨겨요. 버튼 폭은 유지해 레이아웃이 튀지 않게 해요. 로딩 중에는 다시 눌러도 반응하지 않아요.

### 규칙
- **Do** — 라벨은 동사 위주로 짧게 ("확인", "송금하기").
- **Don't** — Primary 버튼을 한 화면에 2개 이상 두지 않아요.

---

## Text Button

배경 없이 텍스트만으로 된 버튼. 리스트 안, 문장 끝의 가벼운 액션에 써요.
- 기본: `blue.500` / 보조: `grey.500` + 오른쪽 chevron("더보기 ›").
- 최소 44px 터치 영역 확보(패딩으로).

## Icon Button

원형 배경에 아이콘 하나. 툴바·헤더의 반복 액션에 써요.
- 크기 44×44px, `radius.full`.
- 중립: 배경 `grey.100` + 아이콘 `grey.700`. 강조: 배경 `blue.500` + 흰 아이콘.

---

## BottomCTA

화면 하단에 고정되어 핵심 행동을 항상 노출하는 버튼 영역.

| 종류 | 구성 |
|---|---|
| **Single** | Primary 버튼 1개 (다음/확인) |
| **Double** | 좌: Secondary, 우: Primary (2:3 비율 권장) |
| **FixedBottomCTA** | 스크롤과 무관하게 하단 고정, 키보드가 올라오면 그 위에 붙음 |

- 하단 safe-area(홈 인디케이터) 여백을 반드시 확보해요.
- 배경은 흰색 + 상단에 옅은 그라데이션/구분선으로 콘텐츠와 분리해요.
