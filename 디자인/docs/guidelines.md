# 작성 가이드라인 (Writing & Do/Don't)

## 문구 톤

- **존댓말 `~해요` 체.** 부드럽고 담백하게. (명령형 `하세요`보다 `~할까요?`, `~했어요`)
- 짧게. 한 문장에 하나의 정보만.
- 전문 용어보다 쉬운 말. ("본인인증" → "본인 확인")
- 숫자·금액은 천 단위 콤마, 단위는 붙여서(`3,200,000원`).

### 예시
| 상황 | Do | Don't |
|---|---|---|
| 완료 토스트 | "링크를 복사했어요" | "복사가 완료되었습니다" |
| 확인 다이얼로그 | "송금을 취소할까요?" | "정말 취소하시겠습니까?" |
| 에러 | "올바른 이메일 형식이 아니에요" | "유효하지 않은 입력값입니다" |

## Do / Don't (전체 공통)

**Do**
- 색·글자·간격은 토큰으로만 참조한다.
- 화면당 Primary 행동은 하나.
- 로딩·빈 상태·에러 상태를 항상 함께 디자인한다.
- 터치 타깃 최소 44×44px.

**Don't**
- HEX·px 하드코딩 금지.
- 강조색 2개 이상 혼용 금지.
- 그라데이션 배경·과한 그림자·불필요한 아이콘/숫자(데이터 슬롭) 금지.
- Toast로 사용자 결정을 요구하지 않기(→ Dialog/BottomSheet).

## 상태 설계 체크리스트

컴포넌트/화면을 만들 때 아래 상태를 빠짐없이 확인해요.

- [ ] 기본 (Default)
- [ ] 로딩 (Loading / Skeleton)
- [ ] 빈 상태 (Empty)
- [ ] 에러 (Error)
- [ ] 비활성 (Disabled)
- [ ] 포커스/눌림 (Focus / Pressed)
- [ ] 접근성 — 더 큰 텍스트, 대비, 44px 타깃

## 접근성

- 본문 텍스트와 배경 대비 최소 4.5:1 (`grey.500` 이상 권장).
- "더 큰 텍스트" 설정에 맞춰 폰트가 커지도록 상대 단위/토큰 사용.
- 색만으로 정보를 전달하지 않기(색 + 아이콘/텍스트 병행).
